# CodeMirror 6 for the time log editor

The time log started as a plain `<textarea>`. Adding syntax highlight required gutter line numbers, line-background tints for parse errors, inline-right read-only widgets for per-entry durations (the active entry's widget ticks every minute), and accent text on the `!` blocker token. A textarea + overlay can do colours and a separate gutter, but it can't put a non-text widget after the typed text on a line without breaking either the editing surface or the visual width — that's the requirement that forced a real editor. CodeMirror 6 has first-class `WidgetType`, `Decoration`, gutter and read-only primitives, and is small enough (~40 KB gzipped per package) to fit a single-screen tool. Lexical/Slate/ProseMirror were considered and rejected as overkill for monospace, single-document editing.

The editor does not re-parse the time log. `parseTimeLog`, `findAllBlockers`, and the per-line duration helper live in `src/core/` and are the source of truth for the rest of the app (submit, billable totals, blocker checks). A CodeMirror `StateField` holds `{ log, error, now, findTicketId }`; React dispatches transactions when any of those change, and the decoration extension is a thin mapping from that field to a `DecorationSet`. This keeps the editor and the rest of the app from drifting apart and avoids re-implementing parsing inside a Lezer grammar.
